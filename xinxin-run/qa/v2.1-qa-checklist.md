# Xinxin v2.1 IP增强验收清单（用于重训后复核）

版本：2.1
适用范围：xinxin-run 目录下所有 row-strip 重生成结果与最终 spritesheet。
触发条件：每次基于 prompts/rows/v2 重新出图后，或修改 `pet_request.json` / `prompts/xinxin-ip-contract.md` 后。

---

## 0. 先决检查
- `imagegen-jobs.json` 中 9 个 row job 的 `prompt_file` 全部指向 `prompts/rows/v2/*.md`
- `pet_request.json` 已配置：
  - `ip_version == "2.1"`
  - `ip_contract_file == "prompts/xinxin-ip-contract.md"`
  - `ip_contract_version == "2.1"`
- `xinxin-warm-lines.zh-CN.json` 版本为 `2.1`

## 1. 资产结构检查
- 解码图是否齐全：
  - decoded/base.png
  - decoded/idle.png
  - decoded/running-right.png
  - decoded/running-left.png
  - decoded/waving.png
  - decoded/jumping.png
  - decoded/failed.png
  - decoded/waiting.png
  - decoded/running.png
  - decoded/review.png
- 帧图目录完整：每个 state 下存在 0 起始序列 PNG，数量 >= 4/5/6/8（对应帧数）
- `final/spritesheet` 是否按最新 decode 生成并可被 QA 查看

## 2. IP身份一致性
- 所有状态均保留：
  - 头发外形与侧边发带/蝴蝶结位置
  - 脸形比例与眼睛样式
  - 服饰主色/细节（粉白分层短裙、丝带、项链、鞋子）
  - 轮廓粗细与像素化边界
  - 5 个基础姿态关键特征（脚部落点、肩位、头位、裙摆层次、配件位置）
- 不允许出现任何新配件、场景元素、Logo、文字、UI、边框
- 色卡上不得出现偏离基底色的 `#00FFFF` 及其相近色

## 3. 帧数与布局检查（逐状态）
- idle：6 帧，6 槽水平单行
- running-right：8 帧，8 槽水平单行
- running-left：8 帧，8 槽水平单行
- waving：4 帧，4 槽水平单行
- jumping：5 帧，5 槽水平单行
- failed：8 帧，8 槽水平单行
- waiting：6 帧，6 槽水平单行
- running：6 帧，6 槽水平单行
- review：6 帧，6 槽水平单行
- 每一帧是否完整、未裁切、未越界、未跨槽位

## 4. 动作连贯性与节奏
- 所有状态首尾帧可无缝衔接（loop 复位）
- 运动幅度小于身份漂移阈值（不出现夸张拉伸、转体翻转、比例跳变）
- waiting/idle 的抖动频率低，running/running-left/right 具备稳定节奏
- failed 的情绪是“轻微失落-自恢复”，不是过度崩坏
- review 不可出现跳帧或过度夸张口部动作

## 5. 风格与渲染禁区
- 禁止高光渐变、光效、阴影滤镜、运动模糊、地面投影、速度线、尾迹、粒子
- 禁止透明空白棋盘/虚假抗锯齿/高细节噪点
- 禁止任何 detached effect（独立漂浮元素）

## 6. 安全回归（可复测）
- 与旧版 `spritesheet.webp` 比较：
  - 不可出现角色换脸/换体型
  - 体态变化仅在“动作”层面，不是“身份”层面
- 如果出现偏差，优先回退到上次稳定输出并只更新失败状态相关 prompt

---

## 7. 一键重跑动作（按 v2 prompt 套件）
1. 仅更新 prompt 或配置后，先重跑指定 row：
   - base（若身份约束变更）
   - idle / running-right / running-left / waving / jumping / failed / waiting / running / review
2. 检查完毕后再重建 final spritesheet 与 qa 输出
3. 通过本清单逐条核验后才进入发布

执行顺序建议：
- baseline：idle, waiting, running
- locomotion：running-right, running-left, jumping
- communication：waving, review
- fallback：failed（最后），随后全量汇总。
