# Xinxin v2.1 快速执行版（3层优先级）

## P0（必须先做，15-30 分钟内）
1. 核对 `pet_request.json`：`ip_version == 2.1` 且 `ip_contract_file == prompts/xinxin-ip-contract.md`
2. 逐项检查 [qa/v2.1-qa-checklist.md](file://D:/pet/xinxin-run/qa/v2.1-qa-checklist.md) 的“先决检查”
3. 先仅重跑 `idle`：
   - 仅当 `idle` 能通过身份一致+6帧+无越界，才继续
4. 用该结果确认颜色、轮廓、背景与身份约束无漂移

## P1（核心状态，按序重跑）
1. 重跑 `waiting`（6帧）
2. 重跑 `running`（6帧）
3. 重跑 `review`（6帧）
4. 重跑 `waving`（4帧）
5. 每个状态重跑后，只做该状态单独核查：
   - 帧数正确
   - 无越界
   - 无异常风格元素

## P2（剩余增强）
1. 重跑 `jumping`（5帧）
2. 重跑 `running-right`（8帧）
3. 重跑 `running-left`（8帧）
4. 重跑 `failed`（8帧）
5. 完成后回查 `running-left` 与 `running-right` 对称性

## 阻塞规则（任何时候都要停）：
- 发现身份漂移（发型/脸型/衣物/配件明显变形）
- 任一状态出现 2 个及以上“槽位/越界”问题
- 出现风格禁区违例（阴影/渐变/文本/UI/漂浮元素）

## 重试路径（发生阻塞后）
1. 先修 `prompts/rows/v2` 中对应 state 的约束段（identity/police rules）
2. 只重跑该状态，不进行全量重跑
3. 复测通过后再继续下一状态

## 通过标准
- 通过 [qa/v2.1-qa-checklist.md](file://D:/pet/xinxin-run/qa/v2.1-qa-checklist.md) 全项后，
- 再执行一次 `frames` 与 `final` 同步导出，
- 最后再按 [qa/v2.1-failure-remediation-playbook.md](file://D:/pet/xinxin-run/qa/v2.1-failure-remediation-playbook.md) 做归档级确认。

## 交付输出（每批次结束）
- `v2` 修订日志（变更 state + 原因）
- 重跑通过快照（decoded/ 和 final 的文件时间戳）
- 最终一句结论：`PASS / RE-TRY`

## 归档与复盘
- 每次执行后按 [run-log-template.md](file://D:/pet/xinxin-run/qa/run-log-template.md) 生成一份记录。
- 建议命名：`run-log-YYYYMMDD-HHMM.md`。
