# 增量 Merge 逻辑

## 任务

用户追加新的飞书聊天记录或回忆时, 把增量信息 merge 进现有的 `memory.md` 和 `persona.md`, 不覆盖已有结论。

## 原则

1. **增量不覆盖**: 新信息追加到对应节末尾, 不替换旧结论。
2. **冲突标注**: 新信息与旧信息矛盾时用 `[⚠️ 冲突]` 标注, 让用户决定。
3. **时间线补充**: 新事件按时间顺序插入时间线。
4. **证据升级**: 新材料能强化旧结论的置信度时, 可以追加 "证据加强:" 行。

## 流程

### 1. 分析增量

按 `memory_analyzer.md` / `persona_analyzer.md` 的维度重新跑一遍, 得到:
- 新事件
- 新的说话风格证据 (新口头禅 / 新 emoji / 标点变化)
- 与旧内容的一致 / 冲突

### 2. Merge 策略

**Memory (事实类)**:
- 新事件 → 插入时间线
- 新共同场景 → 追加到 `## 共同场景`
- 新 inside joke → 追加到 `## Inside Jokes`
- 冲突事实 → `[⚠️ 冲突]` 并列记录, 等用户纠正

**Persona (性格类)**:
- 新语气词 / emoji / 缩写 → 追加到 Layer 2
- 更充分的情绪反应证据 → Layer 3 描述强化
- 新互动模式 → Layer 4
- Layer 0 / 1 不改 (除非用户明确纠正)

### 3. 输出

用 `Edit` 工具追加内容, 新块前标注来源:

```markdown
<!-- [追加于 {日期}, 来源: {feishu / 手动回忆 / 其他}] -->
{新内容}
```

然后重新生成 `SKILL.md`。
