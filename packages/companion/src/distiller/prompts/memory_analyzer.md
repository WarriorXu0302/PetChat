# 共同记忆分析器 (通用联系人版)

## 任务

从原材料中提取你和 ta 的共同上下文, 构建 Shared Context 知识库。
这部分不是"亲密回忆", 而是"让 AI 回复听起来像真的认识用户"的背景池。

## 提取维度

### 1. 认识与关系时间线
- 第一次打交道的时间 / 方式
- 关系的变化 (同事 → 朋友 / 新同事 → 合作了哪几个项目)
- 断联 / 重新联系的节点 (如果有)

### 2. 共同场景
- 共同的工作项目 / 群聊 / 线下场合
- 一起参与过的关键事件 (上线、发版、团建、搬家、转岗...)
- 经常出现的工作/生活话题

### 3. 交流节奏
- 谁更主动? 占比大约多少?
- 高峰时段 (工作日白天? 晚上? 深夜?)
- 消息密度 (每天几条 / 每周几次)
- 主要话题分布 (工作 / 八卦 / 技术 / 生活 / 段子)

### 4. Inside Jokes / 默契
- 两人之间反复出现的梗、外号、暗号
- 常用的固定搭子式的回应 ("收到", "好嘞", 特定表情包...)

### 5. 工作/技术上下文 (如适用)
- 共同使用的工具链、产品名、黑话
- 一起经历的系统/流程/事件
- ta 擅长 / 厌恶的技术或方向

### 6. 冲突与分歧模式
- 合作中出现过的分歧, 谁让步多?
- ta 对什么事会较真?
- 什么情况下你们会进入"客气但冷淡"模式?

### 7. 高光时刻
- ta 帮过你的大事
- 你帮过 ta 的大事
- 共同成就感来源

### 8. 现状
- 现在还在一个团队/公司吗?
- 最近一次聊是什么时候、聊了什么?
- 当前关系强度 (密切合作 / 偶尔联系 / 已经淡)

## 输出格式

```markdown
## 关系时间线
- 认识: {时间, 场景}
- 关系变化: {主要节点}
- 当前状态: {一句话}

## 共同场景
{项目/群/线下场合要点}

## 交流节奏
{主动性 / 频率 / 话题分布}

## Inside Jokes / 默契
{反复出现的梗和暗号}

## 工作/技术上下文
{共同工具链和黑话}

## 分歧与合作模式
{观察到的典型模式}

## 高光时刻
{代表性的协作或帮助记忆}
```

## 注意事项

- 聊天记录中的客观事实优先于用户口述
- 不要美化 ta, 也不要矮化 ta
- 只保留"反复出现"的模式, 不要把单次事件写成人设
- 隐私内容 (薪资、绩效、他人家庭) 只记"存在这个话题", 不记具体内容
